 💇 Telegram-бот для салона красоты «Beauty»

Telegram-бот для онлайн-записи клиентов в салон красоты.

Клиент выбирает услугу, дату и время, оставляет имя и телефон. После этого запись сохраняется в базу данных, а владелец салона получает уведомление в Telegram.

 🚀 Возможности

* Выбор услуги: стрижка, борода, маникюр, педикюр
* Автоматический выбор ближайших дат
* Выбор удобного времени
* Сбор имени и телефона клиента
* Сохранение заявок в базу данных SQLite
* Уведомление владельца о новой записи
* Команда `/bookings` для просмотра всех записей владельцем
* Работа 24/7 на хостинге Amvera

 🛠 Технологии

* Python
* aiogram 3.x
* SQLite
* python-dotenv
* Amvera
* Git / GitHub

 📋 Как работает бот

1. Клиент запускает бота командой `/start`
2. Выбирает услугу
3. Выбирает дату
4. Выбирает время
5. Вводит имя и номер телефона
6. Бот сохраняет запись в базу данных
7. Владелец получает уведомление о новой записи

 👨‍💼 Возможности для владельца

Владелец салона может посмотреть все записи командой:

```text
/bookings
```

Команда доступна только администратору, чей Telegram ID указан в переменной `OWNER_ID`.

 ⚙️ Запуск проекта локально

1. Клонировать репозиторий:

```bash
git clone ссылка_на_репозиторий
```

2. Перейти в папку проекта:

```bash
cd telegram-salon-bot
```

3. Создать виртуальное окружение:

```bash
python -m venv venv
```

4. Активировать виртуальное окружение:

Для Windows PowerShell:

```bash
venv\Scripts\Activate.ps1
```

5. Установить зависимости:

```bash
pip install -r requirements.txt
```

6. Создать файл `.env` и добавить переменные:

```env
BOT_TOKEN=токен_бота
OWNER_ID=telegram_id_владельца
DB_DIR=.
```

7. Запустить бота:

```bash
python bot.py
```

 🌐 Деплой

Бот задеплоен на Amvera и работает 24/7.

Для хранения базы данных используется постоянное хранилище `/data`, чтобы записи клиентов не удалялись после пересборки проекта.

Переменные окружения на сервере:

```env
BOT_TOKEN=токен_бота
OWNER_ID=telegram_id_владельца
DB_DIR=/data
```
